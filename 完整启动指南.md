# 图书管理系统 - 完整启动指南

## 环境准备

确保您已经安装：
- JDK 17 或更高版本 ✓ (您已安装 Java 26)
- Maven 3.8+ ✓ (您已安装)
- Node.js 18+ ✓ (您已安装 Node v24.15.0)
- npm 或 yarn ✓ (您已安装 npm 11.12.1)

---

## 启动步骤

### 第一步：安装前端依赖

```bash
# 在项目根目录 (D:\lenovo\Documents)
npm install
```

### 第二步：解决 Maven 网络问题（如遇到）

如果 Maven 无法下载依赖，可以尝试：

1. 使用国内 Maven 镜像源，在 `library-backend/pom.xml` 中添加：
```xml
<repositories>
    <repository>
        <id>aliyun</id>
        <name>Aliyun Maven</name>
        <url>https://maven.aliyun.com/repository/public</url>
    </repository>
</repositories>
```

2. 或者使用代理

### 第三步：启动后端

```bash
# 进入后端目录
cd library-backend

# 编译并启动（首次会下载依赖）
mvn spring-boot:run
```

后端启动成功后可以访问：
- API 文档: http://localhost:8080/doc.html
- H2 数据库控制台: http://localhost:8080/api/h2-console (JDBC URL: `jdbc:h2:mem:library`, 用户: `sa`, 密码: 留空)

### 第四步：启动前端

打开新的终端窗口：

```bash
# 项目根目录
npm run dev
```

前端启动后访问: http://localhost:5173

## 前后端对接说明

### 前端 API 配置

前端已经配置好 API 地址，在 `.env.development` 文件中：
```
VITE_API_URL=http://localhost:8080/api
```

### 主要 API 接口

认证相关:
- `POST /api/auth/login` - 登录
- `POST /api/auth/register` - 注册
- `POST /api/auth/logout` - 登出
- `GET /api/auth/current` - 获取当前用户

图书管理:
- `GET /api/books` - 获取图书列表
- `GET /api/books/{id}` - 获取图书详情
- `POST /api/books` - 添加图书
- `PUT /api/books/{id}` - 更新图书
- `DELETE /api/books/{id}` - 删除图书

借阅管理:
- `GET /api/borrows/my` - 我的借阅
- `POST /api/borrows` - 借阅
- `PUT /api/borrows/{id}/return` - 归还
- `PUT /api/borrows/{id}/renew` - 续借

### 前端 Context 适配

您可能需要调整前端的 AuthContext 和 BookContext 来使用新的 API 服务：

```typescript
// 在 src/context/AuthContext.tsx 中使用新的 authService
import { authService } from '../services/authService';
```

## 开发建议

1. **先启动后端，再启动前端**
2. 后端 H2 数据库在内存中，重启后数据会重置
3. 如需持久化，可配置 H2 为文件存储或切换到 MySQL

## 常见问题

### Maven 依赖下载失败
- 配置阿里云镜像源
- 检查网络连接

### 前端无法连接后端
- 确认后端已启动在 8080 端口
- 检查 CORS 配置（已配置允许跨域）
- 查看浏览器控制台错误

### Token 过期
- JWT 默认 24 小时有效期
- 重新登录获取新 Token
